## 1. 产品核心理念

### 1.1 设计使命

> **让 AI 能力的发现、获取与使用，像安装 IDE 插件一样简单直观。**

SkillHub Pro 的设计必须始终服务于开发者高效、准确完成工作的诉求。界面不是装饰，而是开发者与海量 AI 技能之间的“智能调度界面”。

### 1.2 三大设计支柱

| 支柱         | 描述                             | 设计启示                             |
| ------------ | -------------------------------- | ------------------------------------ |
| **清晰透明** | 技能内容、来源、安全状态一目了然 | 信息层级分明，关键属性前置，避免隐藏 |
| **高效直接** | 操作路径最短，反馈即时           | 减少弹窗，善用悬浮交互，快捷键友好   |
| **智能可信** | AI 路由决策可解释，结果可预期    | 提供匹配理由，展示备选方案，允许干预 |


## 2. 整体视觉语言

### 2.1 色彩系统

| 色阶        | 色值      | 用途                           |
| ----------- | --------- | ------------------------------ |

| Primary 600 | `#1D4ED8` | 按钮悬浮、按压态               |
| Primary 50  | `#EFF6FF` | 卡片悬浮背景、选中背景         |
| Neutral 900 | `#111827` | 主要文字                       |
| Neutral 600 | `#4B5563` | 辅助文字、图标                 |
| Neutral 200 | `#E5E7EB` | 分割线、边框                   |
| Neutral 50  | `#F9FAFB` | 背景底色                       |

**辅助色**：用于状态反馈

| 颜色   | 色值      | 场景                                 |
| ------ | --------- | ------------------------------------ |
| 成功绿 | `#10B981` | 安装成功、安全状态“safe”             |

### 2.2 字体体系

- **西文**：`Inter`（现代、清晰、屏幕阅读舒适）
- **中文**：`Inter` + 系统默认中文字体回退（`PingFang SC`, `Microsoft YaHei`）
- **等宽代码**：`JetBrains Mono`（用于代码块、SKILL.md 预览）

字号阶梯（基于 16px 基准）：

| 层级    | 字号 | 行高 | 用途           |
| ------- | ---- | ---- | -------------- |
| H1      | 32px | 1.3  | 页面大标题     |
| H2      | 24px | 1.4  | 区块标题       |
| H3      | 20px | 1.4  | 卡片标题       |
| Body L  | 16px | 1.6  | 正文、描述     |
| Body S  | 14px | 1.5  | 辅助信息、标签 |
| Caption | 12px | 1.4  | 脚注、时间戳   |

### 2.3 圆角与间距

- **圆角**：统一使用 `8px` 作为基础圆角，小元素（标签、按钮）可使用 `6px`，大卡片可使用 `12px`。
- **间距系统**：基于 4px 倍数：`4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80`

### 2.4 图标

- **来源**：`Heroicons` (Outline 风格) 或 `Lucide`。
- **尺寸**：16px（内联）、20px（按钮内）、24px（独立功能图标）。
- **用法**：图标始终伴随文字标签（除空间极其受限的工具栏可提供悬浮提示）。


## 3. 布局原则

### 3.1 内容宽度

- 最大内容宽度 `1280px`，左右自动边距，保持舒适的阅读节奏。
- 列表页可采用更宽的 `1440px` 以展示更多列信息。

### 3.2 网格系统

- 采用 12 列栅格，槽宽 `24px`。
- 卡片式布局时，每行列数根据屏幕自适应：Desktop 3-4 列，Tablet 2 列，Mobile 1 列。

### 3.3 信息层级

遵循 **F 型扫描模式**（适合开发者快速浏览）：

- 左上角：Logo + 主导航
- 顶部区域：搜索框（核心交互入口）
- 内容区：左侧筛选栏，右侧结果列表
- 右下角：次要操作（如分页、导出）

## 4. 组件设计原则

### 4.1 按钮

| 类型     | 视觉               | 使用场景                   |
| -------- | ------------------ | -------------------------- |
| 主要按钮 | 填充主色，白色文字 | 页面核心操作（安装、保存） |
| 次要按钮 | 边框 + 主色文字    | 辅助操作（取消、预览）     |
| 文字按钮 | 无边框，主色文字   | 轻量操作（展开、编辑）     |
| 危险按钮 | 红色文字或红色填充 | 删除、卸载                 |

- **最小点击区域**：`32x32px`（移动端 `44x44px`）。
- **状态**：默认、悬浮、按下、禁用、加载中。

### 4.2 卡片

- 使用轻微阴影（`0 2px 8px rgba(0,0,0,0.04)`）和边框（`1px solid #E5E7EB`）区分层次。
- 悬浮时阴影加深（`0 8px 16px rgba(0,0,0,0.08)`），提示可点击。
- 卡片内信息紧凑但不拥挤，留白充足。

### 4.3 搜索框

- 全局搜索框位于导航栏中部，宽度 `400px` ~ `600px`。
- 支持即时搜索（输入 300ms 后自动发起请求），并展示历史记录和热门搜索。
- 搜索结果页提供**语义搜索**切换（调用智能路由），并显示匹配理由（如“基于您描述的数据分析需求推荐”）。

### 4.4 筛选栏

- 采用左侧垂直排列或顶部快速筛选项。
- 筛选项包括：分类、来源、排序方式、安全状态。
- 已选筛选条件以“标签”形式展示在结果上方，支持一键清除。

### 4.5 评分组件

- 5 星评分，支持半星（精度 0.5）。
- 显示平均分和评分人数。

### 4.6 代码预览（SKILL.md）

- 使用深色主题（如 `github-dark`）的语法高亮，提升代码可读性。
- 支持复制内容、全屏查看。
- 若内容过长，默认折叠，提供“展开”按钮。

### 4.7 空状态与加载状态

- **加载中**：骨架屏（Skeleton Screen）而非全屏转圈。
- **空结果**：提供插图 + 引导文案 + 推荐操作（如“浏览热门技能”、“提交技能”）。
- **错误状态**：明确错误原因，并提供重试或反馈入口。


## 5. 交互规范

### 5.1 响应时间与反馈

| 交互类型         | 预期响应时间     | 反馈形式                       |
| ---------------- | ---------------- | ------------------------------ |
| 页面加载         | ≤2s              | 骨架屏                         |
| 点击按钮         | 即时视觉变化     | 按钮 loading 态                |
| 搜索输入         | 300ms 内发起请求 | 输入框右侧显示搜索中状态       |
| 异步操作（安装） | 可能较慢（数秒） | 进度通知（非阻塞），完成后提示 |

### 5.2 导航与面包屑

- 主导航固定顶部，包含：首页、分类、API 文档、关于。
- 二级页面显示面包屑，如：首页 > 数据分析 > 技能名称。

### 5.3 模态框与抽屉

- 尽量少用模态框阻断用户流程。
- 详情预览、设置等可使用右侧抽屉（Drawer），保持上下文。
- 关键确认操作（如删除账号）才使用模态对话框。

### 5.4 快捷键支持

- 全局搜索：`Cmd/Ctrl + K`
- 焦点回到搜索框：`/`
- 详情页快速安装：`Cmd/Ctrl + I`

### 5.5 智能路由交互

- 用户在搜索框输入自然语言时，下方显示“语义匹配中…”状态。
- 匹配结果卡片上展示“匹配度 XX%”和简短理由（如“该技能常用于数据分析场景”）。
- 提供“反馈”入口（👍/👎），用于优化路由模型。


## 6. VS Code 插件设计原则

### 6.1 遵循 VS Code 原生体验

- 界面元素（侧边栏、状态栏、通知）使用 VS Code 标准主题色，自动适配用户选择的主题（深色/浅色）。
- 图标使用 VS Code 内置图标集（`$(icon-name)`）或 `codicon`。

### 6.2 侧边栏视图

- 活动栏图标简洁（建议使用工具箱或拼图概念）。
- 侧边栏分为“发现”和“我的技能”两个标签页。
- 每个技能卡片显示：名称、简介、安装量、安装按钮（或已安装标识）。
- 支持在侧边栏内直接搜索（输入框置顶）。

### 6.3 Webview 详情页

- 点击技能卡片后，在编辑器中打开详情 Webview。
- Webview 内容结构与 Web 端详情页保持一致，但采用 VS Code 主题变量（`--vscode-*`）以确保视觉统一。
- 安装按钮在 Webview 内明显位置，点击后显示进度通知。

### 6.4 本地管理

- “我的技能”列表显示本地已安装技能，展示启用/禁用开关。
- 右键菜单支持：打开文件夹、查看 SKILL.md、卸载、检查更新。

### 6.5 场景化推荐

- 当用户打开特定文件类型（如 `.py`、`.xlsx`）时，状态栏显示推荐技能提示。
- 点击提示可快速安装或执行。


## 7. 移动端适配原则

虽然主要用户为桌面端开发者，但需支持移动端浏览（非核心操作）。

- 导航折叠为汉堡菜单。
- 搜索框占据主要位置。
- 卡片列表改为单列。
- 减少非必要功能（如代码预览默认折叠）。
- 按钮尺寸增大到 `44x44px`。


## 8. 可访问性（A11y）原则

- 所有交互元素必须可通过键盘访问（`Tab` 键切换焦点，`Enter`/`Space` 激活）。
- 焦点指示器清晰可见（使用 `outline` 或主色边框）。
- 图像必须有 `alt` 文本，纯装饰图像设为空。
- 色彩对比度满足 WCAG AA 级（普通文本 ≥4.5:1，大文本 ≥3:1）。
- 表单输入框有明确的标签关联（`<label>`）。
- 支持屏幕阅读器（使用语义化 HTML 和 ARIA 属性）。


## 9. 内容策略与文案

### 9.1 语气

- 专业、清晰、友好，避免技术傲慢。
- 使用主动语态（如“安装技能”而非“技能可被安装”）。
- 错误提示应说明原因并给出解决方案。

### 9.2 统一术语

| 中文      | 英文            | 使用场景     |
| --------- | --------------- | ------------ |
| 技能      | Skill           | 全平台       |
| 安装      | Install         | 获取技能     |
| 启用/禁用 | Enable/Disable  | 插件本地管理 |
| 智能路由  | Smart Router    | 语义匹配功能 |
| 安全状态  | Security Status | 技能安全标记 |

### 9.3 按钮文案

- 使用动词 + 名词，如“安装技能”、“复制命令”、“提交反馈”。
- 避免模糊文案，如“确定”、“好的”。


## 10. 设计资源与交付物

- **设计稿**：Figma 文件，包含设计系统（颜色、字体、组件变体）、关键页面、插件界面。
- **图标集**：统一使用 Heroicons 或 Feather Icons 的 SVG。
- **交互动效**：使用 Lottie 或 CSS 实现简单过渡，时长 `150ms-200ms`。
- **前端实现**：建议采用 Tailwind CSS 快速实现设计系统，插件使用 VS Code Webview UI Toolkit。


## 11. 附录：设计检查清单

在交付设计或前端实现前，请确认以下项：

- [ ] 色彩对比度是否达标？
- [ ] 所有可交互元素是否有悬浮、按下、焦点状态？
- [ ] 移动端窄屏下布局是否正常？
- [ ] 加载状态与空状态是否设计完整？
- [ ] 错误提示是否友好清晰？
- [ ] 插件界面是否与 VS Code 主题自适应？
- [ ] 智能路由匹配结果是否有匹配理由展示？
- [ ] 是否支持键盘主要操作？
