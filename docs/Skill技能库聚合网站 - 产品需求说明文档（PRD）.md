   ## 1. 项目背景与市场分析

   ### 1.1 行业背景

   2026年，AI Agent生态迎来爆发式增长。Anthropic发布的32页Claude Skills实战指南标志着Skills成为AI能力模块化的标准范式——将复杂的工作流程封装为可沉淀、可传播、可复用的数字资产。目前全球已有超过6万个公开可用的Skills，分布在GitHub、专用市场及开发者社区中。头部官方仓库如anthropics/skills已收获8万+ stars，社区仓库awesome-claude-skills突破2.6万stars。

   与此同时，VS Code作为全球开发者首选IDE，月活用户突破5000万，占据73%市场份额。仅AI编程助手类扩展安装量已达百万级别（Claude Code扩展520万+，Cline 500万+），77%的开发者已在使用AI编码工具。这意味着，**IDE已成为AI技能分发的核心战场**——开发者希望在编码环境中直接发现、安装和管理Skills，而非跳出IDE访问Web平台。

   腾讯、字节、阿里等大厂在过去一个月内密集布局Skill分发入口：腾讯推出SkillHub（聚合超1.3万个技能）；字节上线Find Skill（多源聚合GitHub等渠道）；阿里钉钉发布“悟空”平台，目标打造全球最大To B Skill市场。但这些平台均聚焦Web端分发，**IDE插件市场仍存在明显空白**，且缺乏对技能调用的智能调度能力。

   ### 1.2 市场痛点与核心洞察

   1. **技能发现困难**：Skills散落在GitHub上成千上万个仓库中，缺乏统一、高质量的聚合入口。
   2. **技能选择困难**：即使拥有了海量技能，当用户提出一个任务时，AI系统难以从数千个技能中精准选出最合适的那一个。将所有技能描述全量塞入大模型既不经济（上下文窗口限制），也不准确（AI容易产生“选择困难症”）。
   3. **“元数据欺骗”问题**：技能作者编写的简介往往无法完全体现技能的真实能力，仅靠标题和简介匹配可能导致技能选择错误。
   4. **安装与使用门槛高**：普通用户对Skill的安装、配置和使用存在学习曲线，需要更便捷的一键安装和IDE内原生管理体验。
   5. **缺少API调用能力与智能调度**：现有的技能聚合平台多为Web端浏览工具，缺少标准化的RESTful API接口，更缺乏“理解用户意图→智能匹配技能→执行并返回结果”的闭环能力。

   ### 1.3 项目定位

   本项目定位为**一站式AI技能聚合与智能调度平台**，以 **Web平台 + IDE插件 + 开放API + 智能技能路由器** 四位一体架构，核心能力包括：

   - **自动拉取Claude官方技能库**（anthropics/skills等）
   - **自动同步GitHub高星技能仓库**（Stars阈值可配置，默认≥100）
   - **提供标准化RESTful API**供第三方系统调用
   - **提供VS Code插件**，实现IDE内技能发现、一键安装、本地管理、云端同步
   - **构建智能技能路由器（Intelligent Skill Router）**，基于语义理解实现“用户意图→最优技能匹配→上下文组装→执行调度”的完整闭环
   - **采用Golang+Vue3+向量检索+VS Code Extension API现代化架构**

   ## 4. 功能需求

   ### 4.1 技能聚合与同步模块

   #### 4.1.1 官方技能库同步

   - **支持来源**：Anthropic官方skills仓库（github.com/anthropics/skills）、LUNARTECH Superpowers Skills Library等
   - **同步策略**：每日增量同步 + 每周全量同步
   - **解析内容**：Skill名称、描述、分类、作者、版本、依赖关系、SKILL.md全文、YAML元数据
   - **技能结构标准化**：根据SKILL.md标准格式解析，包含YAML frontmatter和Markdown指令正文

   #### 4.1.2 GitHub高星仓库导入

   - **自动发现机制**：基于GitHub Topics标签搜索、文件名精确搜索（`filename:SKILL.md`）、路径搜索（`path:skills`、`path:.claude`）、Awesome列表爬取、Fork网络遍历
   - **质量过滤**：仓库Stars数≥100（可配置阈值）、必须包含SKILL.md文件、自动安全扫描
   - **同步频率**：增量更新每日执行，全量同步每周执行，用户提交仓库按需触发

   #### 4.1.3 数据持久化与管理

   - **主数据库**：PostgreSQL存储技能元数据、分类、用户行为
   - **缓存**：Redis缓存热门技能、搜索结果、API响应
   - **文件存储**：对象存储（OSS/MinIO）存储SKILL.md源文件
   - **向量数据库**：Milvus / Qdrant存储技能的语义向量索引（用于智能路由）

   ### 4.2 技能浏览与检索模块（前端Vue3）

   #### 4.2.1 首页功能

   - 技能总量统计（实时更新）
   - 热门技能排行榜（综合GitHub Stars、下载量、评分、路由调用量）
   - 最新收录技能时间线
   - 分类快捷入口
   - 精选技能推荐

   #### 4.2.2 技能搜索与筛选

   - **全文搜索**：基于Meilisearch/Elasticsearch，支持模糊匹配和拼音搜索
   - **语义搜索（智能路由联动）**：提供“自然语言描述需求”的搜索框，调用智能路由的检索能力返回最相关技能
   - **多维筛选**：按分类、来源、Stars数、安全状态筛选

   #### 4.2.3 技能详情页

   - 技能基本信息展示（名称、作者、Stars、分类、版本）
   - SKILL.md在线预览（语法高亮）
   - 文件目录结构展示
   - 安装指令一键复制
   - **“在VS Code中安装”按钮**（唤起插件）
   - **“试用此技能”交互式Demo**（调用路由API，输入示例任务返回执行结果预览）
   - 相似技能推荐（基于向量相似度）
   - 评分与评论区域（阶段二）
   - 收藏功能（阶段二）

   ### 4.5 智能技能路由器模块（新增核心模块）

   #### 4.5.1 模块背景与定位

   随着平台聚合的技能数量增长，用户面临的核心痛点从“找不到技能”转变为“不知道用哪个技能”。智能技能路由器旨在解决：

   - **上下文窗口限制**：无法将所有技能描述全量塞入LLM上下文。
   - **AI选择困难**：选项过多导致LLM匹配准确率下降。
   - **元数据欺骗**：仅靠技能简介无法体现真实能力，容易误匹配。

   路由器通过 **语义索引 + 粗排召回 + 精排重排序 + 上下文组装** 的四层架构，实现从用户自然语言意图到最优技能匹配与执行的完整闭环。

   #### 4.5.2 路由工作流程

   ```mermaid
   graph TD
       A[用户输入自然语言指令] --> B{API Gateway};
       B --> C[意图识别与检索服务];
       C --> D[(向量数据库<br/>Milvus/Qdrant)];
       C --> E[(全文检索引擎<br/>Meilisearch)];
       D --> F[粗排候选集<br/>Top 50];
       E --> F;
       F --> G[重排序服务<br/>Cross-Encoder Reranker];
       G --> H[精排结果<br/>Top 3];
       H --> I[上下文组装服务];
       I --> J[(对象存储<br/>读取SKILL.md)];
       J --> K[发送至LLM执行];
       K --> L[返回执行结果];
   ```

   #### 4.5.3 核心子模块设计

   ##### 4.5.3.1 语义索引构建模块

   在技能同步入库时触发：

   - **输入**：技能的`SKILL.md`全文、元数据（名称、描述、分类、标签）
   - **处理**：
     1. 使用**LLM辅助特征提取**：调用大模型（如GPT-4o-mini或Claude Haiku）自动从技能文档中提取关键步骤、核心能力、适用场景、前置条件、输入输出格式。生成一段“增强描述文本”。
     2. 将“名称+描述+增强描述文本”拼接，通过Embedding模型（如`bge-m3`或`text-embedding-3-small`）转化为向量。
     3. 将向量与技能ID、元数据存入向量数据库。
   - **存储结构**：
     - 向量字段：`embedding`（1536维或1024维）
     - 标量字段：`skill_id`、`name`、`category`、`source_type`、`github_stars`
     - 全文索引字段：`enhanced_description`（用于混合检索）

   ##### 4.5.3.2 意图识别与混合检索（粗排）

   - **输入**：用户自然语言查询（如“帮我分析这个Excel表格里的销售趋势，并生成一份PPT报告”）
   - **处理**：
     1. **语义向量检索**：将查询文本转化为向量，在向量数据库中执行ANN（近似最近邻）搜索，返回Top 50相似技能。
     2. **关键词检索**：同时使用Meilisearch对查询进行分词匹配，返回Top 50相关技能。
     3. **融合排序**：使用**倒数排名融合（RRF）** 算法合并两个结果集，生成最终粗排候选集（Top 50）。
   - **输出**：候选技能ID列表（带初步相关性分数）。

   ##### 4.5.3.3 重排序（精排）

   - **输入**：粗排候选集（Top 50）及用户原始查询。
   - **处理**：
     1. 对每个候选技能，构造`[查询, 技能增强描述]`文本对。
     2. 送入**Cross-Encoder重排序模型**（如`bge-reranker-v2-m3`或Cohere Rerank API），计算深度语义匹配分数。
     3. 根据重排序分数降序排列，取Top 3作为最终推荐技能。
   - **输出**：精排技能ID列表（含匹配分数）。

   ##### 4.5.3.4 上下文组装与指令压缩

   - **输入**：精排Top 3技能ID、用户原始查询。
   - **处理**：
     1. 从对象存储读取对应技能的`SKILL.md`全文。
     2. （可选）若技能文档过长，调用**指令压缩模型**（如基于LLM的摘要或SkillReducer算法）提取核心指令部分。
     3. 按预设模板组装提示词：
        ```
        [系统指令]
        你是一个AI助手，可以使用以下技能来完成任务。请根据用户指令选择并执行最合适的技能。
        
        [可用技能]
        ## 技能1: {name}
        {description}
        {compressed_instructions}
        
        ## 技能2: ...
        
        [用户指令]
        {user_query}
        ```
     4. 将组装好的上下文发送给执行LLM（可配置，支持Claude、GPT、DeepSeek等）。
   - **输出**：LLM执行结果（或技能执行计划）。

   ##### 4.5.3.5 执行结果反馈与迭代优化

   - **记录数据**：记录用户查询、最终选择的技能ID、用户对结果的满意度（通过API返回值中的`feedback`字段收集）。
   - **离线优化**：使用收集的反馈数据对重排序模型进行微调，持续提升匹配准确率。

   #### 4.5.4 智能路由API详细设计

   ##### 4.5.4.1 匹配接口（`POST /api/v1/router/match`）

   **功能**：仅返回匹配的技能信息，不执行技能。适用于用户希望先预览技能再决定是否执行的场景。

   **请求体**：
   ```json
   {
     "query": "帮我分析这个Excel表格里的销售趋势",
     "top_k": 3,
     "include_skill_content": false
   }
   ```

   **响应体**：
   ```json
   {
     "code": 0,
     "data": {
       "matches": [
         {
           "skill_id": "uuid",
           "name": "excel-trend-analyzer",
           "description": "分析Excel数据趋势并生成图表",
           "match_score": 0.95,
           "skill_md_url": "https://..."
         }
       ],
       "query_embedding_time_ms": 120,
       "rerank_time_ms": 45
     }
   }
   ```

   ##### 4.5.4.2 执行接口（`POST /api/v1/router/execute`）

   **功能**：端到端执行：意图→技能匹配→上下文组装→LLM执行→返回结果。

   **请求体**：
   ```json
   {
     "query": "帮我分析这个Excel表格里的销售趋势，并生成一份PPT报告",
     "execution_config": {
       "model": "claude-3.5-sonnet",
       "max_tokens": 4096,
       "temperature": 0.2,
       "stream": false
     },
     "context": {
       "user_id": "optional",
       "session_id": "optional"
     }
   }
   ```

   **响应体**：
   ```json
   {
     "code": 0,
     "data": {
       "result": "根据分析，销售额在过去6个月呈现上升趋势...（完整执行结果）",
       "selected_skills": [
         {
   ```

   ### 6.3 核心模块设计

   #### 6.3.1 技能同步爬虫模块

   ```go
   type SkillCrawler interface {
       SyncOfficialRepo(ctx context.Context) error
       DiscoverAndSyncGitHub(ctx context.Context, minStars int) error
       SyncRepository(ctx context.Context, repoURL string) error
   }
   ```

   #### 6.3.2 智能路由器服务接口

   ```go
   type SkillRouter interface {
       // 匹配技能（仅返回匹配结果）
       Match(ctx context.Context, query string, topK int) ([]SkillMatch, error)
       // 执行技能（端到端）
       Execute(ctx context.Context, req *ExecuteRequest) (*ExecuteResponse, error)
       // 重建向量索引（管理员触发）
       RebuildIndex(ctx context.Context) error
   }
   
   type SkillMatch struct {
       SkillID     string
       Name        string
       Description string
       Score       float64
       Metadata    map[string]interface{}
   }
   
   type ExecuteRequest struct {
       Query           string
       Model           string
       MaxTokens       int
       Temperature     float32
       Stream          bool
       UserContext     map[string]interface{}
   }
   
   type ExecuteResponse struct {
       Result         string
       SelectedSkills []SelectedSkill
       Meta           ExecutionMeta
   }
   ```

   #### 6.3.3 向量索引构建服务

   ```go
   type EmbeddingService interface {
       // 为技能生成增强描述
       GenerateEnhancedDescription(ctx context.Context, skillMD string) (string, error)
       // 生成向量
       Embed(ctx context.Context, text string) ([]float32, error)
       // 批量索引技能
       IndexSkills(ctx context.Context, skills []*Skill) error
   }
   ```

   #### 6.3.4 数据模型扩展

   ```sql
   -- 在原有skills表基础上新增字段
   ALTER TABLE skills ADD COLUMN enhanced_description TEXT;
   ALTER TABLE skills ADD COLUMN embedding_version VARCHAR(20);
   ALTER TABLE skills ADD COLUMN last_embedded_at TIMESTAMP;
   
   -- 路由调用日志表（用于分析和优化）
   CREATE TABLE router_logs (
       id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
       user_id UUID REFERENCES users(id) ON DELETE SET NULL,
       query TEXT NOT NULL,
       matched_skill_ids JSONB,
       selected_skill_id UUID,
       execution_success BOOLEAN,
       user_feedback SMALLINT,  -- 1-5分

   CREATE INDEX idx_router_logs_user ON router_logs(user_id);
   ```
