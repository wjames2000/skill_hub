## 1. WBS 分解原则

本 WBS 基于以下原则构建：

- **面向交付**：每个工作包对应一个可独立验收的交付物。
- **层级分解**：阶段 → 模块 → 功能 → 开发任务。
- **100% 覆盖**：覆盖从项目启动到上线的全部工作。
- **工时估算**：基于 1 人天 = 8 有效工作小时，已考虑技术调研和联调时间。

**总项目工期**：约 13 周（含缓冲）

## 4. 关键路径与里程碑

基于各任务依赖关系，关键路径如下：

```
项目启动 → 后端基础设施 → 技能同步 → 向量化 → 智能路由 API → 业务 API → 前端/插件对接 → 测试 → 上线
```

| 里程碑                     | 计划完成时间 | 验收标准                    |
| -------------------------- | ------------ | --------------------------- |
| M1：启动阶段完成           | 第 2 周末    | 环境就绪，设计评审通过      |
